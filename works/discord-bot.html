Discord Bot 開発（複数プロジェクト）

Discordサーバー運営を効率化するためのBotを設計・開発。
フォーム入力・権限管理・外部ファイル操作などを自動化し、運営負荷を低減しました。

種別 個人開発
期間 2024/09〜現在
担当 要件整理 / 設計 / 実装 / 運用

概要

  • 目的：Discordサーバー運営における手作業や人的ミスを削減し、安定運用を実現するため
  • 成果：選手登録・権限管理・Linuxサーバー上の運用作業（ファイル操作等）をBotで自動化
  • 利用シーン：ゲーム大会運営、コミュニティ管理、VPS上のLinuxサーバー運用支援

画像準備中・・・

使用技術

  • JavaScript (Node.js)
  • Discord.js v14
  • JSON / File I/O
  • Linux (VPS)
  • screen / systemd

機能

  • モーダルフォームを用いた情報入力（名前・URL・画像など）
  • ボタン操作による処理分岐・状態管理
  • ロールに基づく権限チェックと操作制限
  • Discord経由でのVPS上ファイル編集（例：Whitelist管理）
    └ サーバーへ直接接続できないメンバーでも運用作業が可能
  • Linux操作が分からない人でも扱える運用導線（ボタン・フォーム中心）
    └ 運営の技術力に依存せず稼働できる構成を意識
  • ミニゲームBot（ブラックジャック等）の実装

工夫点

  • Discord.js v14のInteraction構造を前提に、 コマンド・ボタン・モーダルを役割ごとに分離した設計を採用
  • 操作単位で権限チェックを行い、 誤操作や不正操作を防止する設計
  • Linuxサーバー運用を「Discord上の操作」に置き換え、 サーバーへの接続権限やLinux知識がなくても運用できるようにした
  • ファイル操作処理をサービス層として分離し、 Bot本体の保守性・拡張性を向上
